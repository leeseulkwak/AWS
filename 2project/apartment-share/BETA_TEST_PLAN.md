# 베타 테스트 실험 계획서

## 1. 서비스 정의

**서비스명**: 동네빌려

**서비스 개요**: 같은 아파트 단지 또는 인근 주민 간 물건을 단기간 대여할 수 있는 커뮤니티 기반 공유 서비스

### 핵심 가치

1. **즉시 해결**: 급하게 필요한 물건을 즉시 해결
2. **비용 절감**: 구매 대신 대여로 경제적 소비
3. **커뮤니티 활성화**: 이웃 간 신뢰와 공유 문화 형성

### 대상 품목

- **초기 베타**: 공구/DIY 도구, 캠핑 장비 (손상 리스크 낮음)
- **향후 확장**: 주방기기, 유아용품, 스포츠 용품, 전자제품

## 2. 차별화 요소

### 입주민 인증 시스템
- 아파트 주민 인증을 통한 신뢰도 확보
- 같은 단지 내 거래로 접근성 향상

### 비대면 거래 시스템
- **문고리 거래**: 현관 문고리에 걸어두기
- **무인택배함**: 무인택배함 이용
- **관리사무소**: 관리사무소 맡기기
- 직접 대면 없이도 안전한 거래 가능

### 사진 인증 시스템
- 수령 시: 물건 상태 촬영 및 업로드
- 반납 시: 물건 상태 촬영 및 업로드
- 쏘카 방식의 상태 관리로 파손/분실 분쟁 최소화

### 보험 적용
- 삼성화재 보험 연계
- 파손 및 분실 시 보상 체계 마련
- 안심하고 이용할 수 있는 환경 제공

## 3. 비즈니스 모델

### 수익원

1. **거래 수수료**: 거래 금액의 5-10%
2. **보험료**: 거래 금액의 2-3%
3. **마케팅 제휴**: 제조사/판매사 제휴 수수료
4. **프리미엄 기능**: 우선 대여, 확장 보험 등

### 확장 가능성

**"대여하여 써보니 어때요? 구매하러 가볼까요?"**
- 대여 후 구매 전환 시 할인 혜택
- 제조사/판매사와 제휴를 통한 수익 모델
- 체험 마케팅 채널로 활용

## 4. 베타 테스트 설계

### 테스트 대상

- **아파트 수**: 3~5개 대표 아파트
- **목표 사용자**: 각 아파트당 100-200명
- **테스트 기간**: 8주 (2개월)
- **지역**: 서울/수도권 신축 아파트 중심

### 테스트 단계

#### Phase 1: 초기 온보딩 (1-2주)
- 입주민 인증 및 가입
- 초기 물건 등록 (20-30개)
- 사용법 교육 및 안내

#### Phase 2: 활성화 (3-5주)
- 거래 촉진 이벤트
- 리뷰 작성 유도
- 커뮤니티 형성

#### Phase 3: 안정화 (6-8주)
- 자발적 거래 유도
- 데이터 수집 및 분석
- 사용자 피드백 수렴

## 5. 핵심 KPI 설정

### 사용자 활성도 지표

| KPI | 목표 | 측정 방법 |
|-----|------|----------|
| DAU (일일 활성 사용자) | 전체 사용자의 20% | 일일 로그인 사용자 수 |
| MAU (월간 활성 사용자) | 전체 사용자의 60% | 월간 로그인 사용자 수 |
| 재방문율 | 70% 이상 | 2회 이상 이용한 사용자 비율 |

### 거래 지표

| KPI | 목표 | 측정 방법 |
|-----|------|----------|
| 거래 성공률 | 90% 이상 | 완료된 거래 / 신청된 거래 |
| 평균 대여 기간 | 2-3일 | 총 대여일 / 총 거래 건수 |
| 거래 완료율 | 95% 이상 | 정상 반납 / 전체 거래 |
| 주간 거래 건수 | 20건 이상 | 주당 완료된 거래 수 |

### 커뮤니티 지표

| KPI | 목표 | 측정 방법 |
|-----|------|----------|
| 이웃 간 거래 비율 | 80% 이상 | 같은 동 거래 / 전체 거래 |
| 리뷰 작성률 | 85% 이상 | 작성된 리뷰 / 완료된 거래 |
| 평균 평점 | 4.5점 이상 | 전체 리뷰 평점 평균 |

### 비즈니스 지표

| KPI | 목표 | 측정 방법 |
|-----|------|----------|
| 평균 거래액 | 5,000-10,000원 | 총 거래액 / 거래 건수 |
| 보험 청구율 | 5% 이하 | 보험 청구 건수 / 전체 거래 |
| 물건 파손율 | 3% 이하 | 파손 신고 건수 / 전체 거래 |

### 확장성 지표

| KPI | 목표 | 측정 방법 |
|-----|------|----------|
| 물건 등록 증가율 | 월 20% 이상 | 월별 신규 등록 물건 수 증가 |
| 구매 전환율 | 10% 이상 | 대여 후 구매한 사용자 비율 |
| NPS (순추천지수) | 50 이상 | 추천 의향 설문 |

## 6. 실험 설계

### A/B 테스트 항목

1. **가격 정책**
   - A안: 물건 주인이 자유롭게 가격 설정
   - B안: 카테고리별 권장 가격 제시

2. **수령 방법**
   - A안: 문고리 거래 중심
   - B안: 무인택배함 중심

3. **보험료 책정**
   - A안: 거래액의 2%
   - B안: 거래액의 3%

### 사용자 인터뷰

- **주간 인터뷰**: 10명 (활발한 사용자 5명 + 비활성 사용자 5명)
- **질문 항목**:
  - 서비스 이용 동기
  - 불편한 점
  - 개선 제안
  - 지불 의향 가격

## 7. 리스크 관리

### 예상 리스크

| 리스크 | 대응 방안 |
|--------|----------|
| 물건 파손/분실 | 삼성화재 보험, 사진 인증 시스템 |
| 사용자 간 분쟁 | 고객센터 운영, 분쟁 해결 가이드 |
| 낮은 참여율 | 초기 인센티브 제공, 이벤트 |
| 보험 사기 | 사진 인증 강화, 이력 관리 |

### 중단 기준

다음 조건 중 2개 이상 충족 시 베타 테스트 중단 검토:
- 거래 성공률 70% 미만
- 물건 파손율 10% 이상
- 보험 청구율 15% 이상
- MAU 30% 미만 지속

## 8. 데이터 수집 계획

### 정량 데이터
- 사용자 행동 로그 (페이지 방문, 클릭, 검색)
- 거래 데이터 (신청, 승인, 완료, 취소)
- 리뷰 데이터 (평점, 내용, 작성 시간)
- 물건 데이터 (카테고리, 대여 빈도, 가격)

### 정성 데이터
- 사용자 인터뷰 (주간)
- 리뷰 텍스트 분석
- 고객센터 문의 내용
- 커뮤니티 반응

## 9. 성공 기준

### 최소 성공 기준 (MVP)
- MAU 50% 이상
- 거래 성공률 85% 이상
- 파손율 5% 이하
- NPS 30 이상

### 목표 성공 기준
- MAU 60% 이상
- 거래 성공률 90% 이상
- 파손율 3% 이하
- NPS 50 이상

### 확장 조건
목표 성공 기준 달성 시:
- 10개 아파트로 확장
- 고가 품목 카테고리 추가
- 제조사/판매사 제휴 시작

## 11. 프로토타입 화면 구성

### 구현된 화면
1. **홈**: 물건 탐색 및 검색, 카테고리 필터
2. **물건 상세**: 대여 신청, 수령 방법 선택
3. **내 대여**: 대여/빌림 관리, 사진 인증
4. **KPI 대시보드**: 실시간 지표 시각화
5. **마이페이지**: 사용자 정보 및 통계

### 기술 스택
- React 18 + TypeScript
- Vite (빠른 개발)
- TailwindCSS (UI)
- Chart.js (시각화)
- React Router (라우팅)
